## 一、项目现状分析

### 1.1 技术栈概览

| 组件 | 版本/信息 |

| 向量存储 | SimpleVectorStore (默认) / PgVectorStore (可选) |
| 记忆系统 | JDBC Chat Memory (H2 默认, PostgreSQL 可选) |
| HTTP 客户端 | 自定义 OkHttp (含重试逻辑) |

### 1.2 现有 AI 交互架构

```
[ChatController] POST /api/chat (JSON)
       ↓
[ChatMessage{role, content, conversationId}]
       ↓
[AgentService]
   ├── SkillsAdvisor (技能系统注入)
   ├── MessageChatMemoryAdvisor (JDBC 消息窗口)
   ├── VectorStoreChatMemoryAdvisor (语义记忆)
   ├── QuestionAnswerAdvisor (RAG 知识库)
   ├── SkillTools (默认工具)
   └── ChatClient (LLM 调用)
```

### 1.3 现有组件清单

| 组件 | 路径 | 功能 |
|------|------|------|
| ChatController | `controller/ChatController.java` | HTTP 入口，接收 JSON ChatMessage |
| AgentService | `service/AgentService.java` | 核心聊天逻辑，多 Advisor 编排 |
| SpringAiConfig | `config/SpringAiConfig.java` | ChatClient/ChatModel 手动配置 |
| ChatMessage | `model/ChatMessage.java` | 消息 DTO (仅文本) |
| SkillsAdvisor | `agent/SkillsAdvisor.java` | 技能系统 prompt 注入 |
| SkillTools | `agent/SkillTools.java` | 技能执行工具 |

### 1.4 当前限制

- **仅支持文本输入**：ChatMessage.content 为 String 类型
- **无图片处理能力**：无 Media 类型支持
- **无语音处理能力**：无 TranscriptionModel 配置
- **无多模态 ChatClient**：现有 ChatClient 仅处理纯文本

### 1.5 项目特性对方案的影响

1. **自定义 OkHttp 配置**：项目使用自定义 OkHttpClient 而非 Spring AI 自动装配，需要确保多模态请求也能复用现有重试逻辑
2. **多 Provider 支持**：已实现 OpenAI/Anthropic 切换机制，需要考虑多模态场景下的 Provider 扩展
3. **现有 Advisor 架构**：AgentService 使用多层 Advisor，设计时需要考虑多模态输入如何融入现有架构

## 二、设计目标

### 2.1 功能目标

| 目标 | 描述 | 优先级 |
|------|------|--------|
| 图片输入支持 | 用户可上传图片，结合文本或单独上传 | P0 |
| 语音输入支持 | 用户可上传语音文件，自动转写为文本后再处理 | P0 |
| 现有功能兼容 | 纯文本对话功能保持不变 | P0 |
| RAG 集成 | 多模态输入可结合现有知识库检索 | P0 |
| 低成本改造 | 最小化对现有架构的侵入 | P1 |

### 2.2 非功能目标

- **可切换性**：支持切换到不支持多模态的主模型（如 MiniMax M2.7），通过视觉模型代理
- **配置简单化**：通过配置文件切换多模态/非多模态模式
- **低风险**：新增代码与现有代码解耦，不影响现有功能

### 2.3 使用场景

1. **纯文本对话**（兼容现有）：`POST /api/chat/text`
2. **文本 + 图片**：`POST /api/chat` with multipart
3. **文本 + 语音**：`POST /api/chat` with multipart
4. **图片 + 语音 + 文本**：`POST /api/chat` with multipart

## 三、详细设计方案

### 3.1 方案：双模型协作方案（主模型 + 视觉模型）

**选定方案**：采用双模型协作方案作为主要实现方案

**适用场景**：
- 主模型不支持多模态（如 MiniMax M2.7）
- 或希望分离视觉理解和文本生成的职责以优化成本

**架构图**：
```
用户 [图片]  → [visionChatClient(视觉模型)] → 图片描述文字
用户 [音频]  → [TranscriptionModel(ASR模型)] → 转写文字
用户 [文本]  → 直接保留
                    ↓ 三路文字合并
              [RAG 检索]
                    ↓
              [chatClient(主模型)] → 回复
```

**设计原则**：
1. 保持现有 `chatClient` 命名不变，这是主模型对应的 ChatClient
2. 新增 `visionChatClient` 用于视觉模型（图片理解）
3. `TranscriptionModel` 用于语音转文字

**关于 visionChatClient 的配置说明：**

双模型方案需要在 `SpringAiConfig.java` 中新增 `visionChatClient` Bean，用于视觉模型（图片理解）。现有 `chatClient` Bean 保持不变。

```java
@Bean("visionChatClient")
public ChatClient visionChatClient() {
    // 视觉模型的配置（使用视觉模型 provider，如火山方舟/豆包视觉）
    OpenAiApi visionApi = OpenAiApi.builder()
            .baseUrl(visionBaseUrl)   // 视觉模型 base-url
            .apiKey(visionApiKey)     // 视觉模型 api-key
            .build();

    OpenAiChatModel visionChatModel = OpenAiChatModel.builder()
            .openAiApi(visionApi)
            .defaultOptions(OpenAiChatOptions.builder()
                    .model(visionModel)  // 视觉模型名称
                    .build())
            .build();

    return ChatClient.create(visionChatModel);
}
```

**关于 TranscriptionModel 配置：**

如果 `TranscriptionModel` 未能自动装配，需要手动添加 Bean：

#### 4.3.3 SpringAiMultimodalChatService 实现

```java
// SpringAiMultimodalChatService.java
package com.example.demo.service.impl;

import com.example.demo.service.MultimodalChatService;
import com.example.demo.service.RagService;
import org.springframework.ai.audio.transcription.TranscriptionModel;
import org.springframework.ai.chat.client.ChatClient;
import org.springframework.ai.chat.memory.ChatMemory;
import org.springframework.ai.content.Media;
import org.springframework.beans.factory.annotation.Qualifier;
import org.springframework.core.io.Resource;
import org.springframework.http.MediaType;
import org.springframework.stereotype.Service;

@Service
public class SpringAiMultimodalChatService implements MultimodalChatService {

    private final ChatClient chatClient;                    // 主 ChatClient（现有命名，不修改）
    private final ChatClient visionChatClient;              // 视觉模型 ChatClient
    private final TranscriptionModel transcriptionModel;    // 语音转写模型
    private final RagService ragService;                    // RAG 服务

    public SpringAiMultimodalChatService(
            ChatClient chatClient,
            @Qualifier("visionChatClient") ChatClient visionChatClient,
            TranscriptionModel transcriptionModel,
            RagService ragService) {
        this.chatClient = chatClient;
        this.visionChatClient = visionChatClient;
        this.transcriptionModel = transcriptionModel;
        this.ragService = ragService;
    }

    @Override
    public String chat(String query,
                      String instruction,
                      String conversationId,
                      Resource image,
                      String imageContentType,
                      Resource audio) {

        StringBuilder userTextBuilder = new StringBuilder();

        // 1. 保留用户原始文本
        if (query != null && !query.isBlank()) {
            userTextBuilder.append("用户文本: ").append(query).append("\n");
        }

        // 2. 语音转写（如果提供了音频）
        if (audio != null) {
            String transcription = transcriptionModel.transcribe(audio);
            userTextBuilder.append("语音转写: ").append(transcription).append("\n");
        }

        // 3. 图片处理 - 通过视觉模型获取描述（双模型方案）
        if (image != null) {
            String imageDescription = describeImage(image, imageContentType, query);
            userTextBuilder.append("图片内容描述: ").append(imageDescription).append("\n");
        }

        String userText = userTextBuilder.toString();
        if (userText.isBlank()) {
            userText = "用户未提供有效输入。";
        }

        // 4. RAG：基于综合后的文字内容做知识检索
        String context = ragService.buildContext(userText);

        // 5. 构造 prompt
        String systemPrompt = instruction != null ? instruction :
            "你是一个知识库问答助手。请严格基于提供的知识上下文回答用户问题。";

        String finalUserText = """
            [知识上下文]
            %s

            [用户输入]
            %s
            """.formatted(context, userText);

        // 6. 调用主 ChatClient（纯文本，不传图片）
        return chatClient.prompt()
                .system(systemPrompt)
                .user(finalUserText)
                .advisors(a -> a.param(ChatMemory.CONVERSATION_ID, conversationId))
                .call()
                .content();
    }

    /**
     * 使用视觉模型将图片转换为文字描述
     */
    private String describeImage(Resource image, String imageContentType, String hint) {
        MediaType mt = parseMediaType(imageContentType, MediaType.IMAGE_JPEG);
        Media media = new Media(mt, image);

        // 如果用户有文本提示，一并告诉视觉模型，让描述更有针对性
        String visionPrompt = (hint != null && !hint.isBlank())
                ? "用户问题是：" + hint + "\n请详细描述这张图片的内容，包括文字、数据、图表、场景等所有重要信息。"
                : "请详细描述这张图片的内容，包括文字、数据、图表、场景等所有重要信息。";

        return visionChatClient.prompt()
                .user(user -> user.text(visionPrompt).media(media))
                .call()
                .content();
    }

    private MediaType parseMediaType(String contentType, MediaType defaultType) {
        if (contentType == null || contentType.isBlank()) {
            return defaultType;
        }
        try {
            return MediaType.parseMediaType(contentType);
        } catch (Exception e) {
            return defaultType;
        }
    }
}
```

#### 4.3.5 ExistingRagService 实现（封装现有逻辑）

```java
// ExistingRagService.java
package com.example.demo.service.impl;

import com.example.demo.service.RagService;
import org.springframework.ai.vectorstore.VectorStore;
import org.springframework.ai.document.Document;
import org.springframework.stereotype.Service;

import java.util.List;
import java.util.stream.Collectors;

/**
 * 现有 RAG 逻辑的封装
 * 对接 AgentService 中已有的 RAG 实现
 *
 * 说明：本实现直接调用 VectorStore 的相似性搜索功能，
 * 不依赖 QuestionAnswerAdvisor，以保持轻量级实现。
 * 如需更复杂的 RAG 逻辑（如重新排序、分块策略等），
 * 可自行扩展本实现。
 */
@Service
public class ExistingRagService implements RagService {

    private final VectorStore vectorStore;

    public ExistingRagService(VectorStore vectorStore) {
        this.vectorStore = vectorStore;
    }

    @Override
    public String buildContext(String userQuery) {
        // 检索与用户输入最相关的文档
        List<Document> documents = vectorStore.similaritySearch(userQuery);

        if (documents.isEmpty()) {
            return "【知识库无相关内容】";
        }

        // 将检索到的文档格式化为上下文字符串
        return documents.stream()
                .map(doc -> doc.getContent())
                .collect(Collectors.joining("\n\n"));
    }
}
```

**重要说明：**
- 本实现直接使用 `VectorStore.similaritySearch()` 获取相关文档
- 如果需要更复杂的 RAG 逻辑（如混合检索、重排序、多跳推理等），可以在此基础上扩展
- 现有的 `QuestionAnswerAdvisor` 提供了更完善的 RAG 逻辑，如果需要复用，可以参考 `AgentService` 中的实现方式

#### 4.3.6 MultimodalChatController

```java
// MultimodalChatController.java
package com.example.demo.controller;

import com.example.demo.dto.MultimodalChatRequest;
import com.example.demo.dto.MultimodalChatResponse;
import com.example.demo.service.MultimodalChatService;
import org.springframework.core.io.FileSystemResource;
import org.springframework.http.MediaType;
import org.springframework.web.bind.annotation.*;
import org.springframework.web.multipart.MultipartFile;

import java.io.File;

@RestController
@RequestMapping("/api/chat")
public class MultimodalChatController {

    private final MultimodalChatService chatService;

    public MultimodalChatController(MultimodalChatService chatService) {
        this.chatService = chatService;
    }

    /**
     * 多模态聊天入口
     * 支持同时上传文本、图片、音频
     */
    @PostMapping(consumes = MediaType.MULTIPART_FORM_DATA_VALUE)
    public MultimodalChatResponse chat(
            @RequestParam(value = "query", required = false) String query,
            @RequestParam(value = "instruction", required = false) String instruction,
            @RequestParam(value = "conversationId", required = false, defaultValue = "default") String conversationId,
            @RequestPart(value = "image", required = false) MultipartFile image,
            @RequestPart(value = "audio", required = false) MultipartFile audio
    ) throws Exception {

        File imageTemp = null;
        File audioTemp = null;

        try {
            // 处理图片
            FileSystemResource imageResource = null;
            String imageContentType = null;
            if (image != null && !image.isEmpty()) {
                imageTemp = File.createTempFile("img-", "-" + image.getOriginalFilename());
                image.transferTo(imageTemp);
                imageResource = new FileSystemResource(imageTemp);
                imageContentType = image.getContentType();
            }

            // 处理音频
            FileSystemResource audioResource = null;
            if (audio != null && !audio.isEmpty()) {
                audioTemp = File.createTempFile("audio-", "-" + audio.getOriginalFilename());
                audio.transferTo(audioTemp);
                audioResource = new FileSystemResource(audioTemp);
            }

            String answer = chatService.chat(
                    query,
                    instruction,
                    conversationId,
                    imageResource,
                    imageContentType,
                    audioResource
            );

            return new MultimodalChatResponse(answer);

        } finally {
            // 清理临时文件
            if (imageTemp != null && imageTemp.exists()) {
                imageTemp.delete();
            }
            if (audioTemp != null && audioTemp.exists()) {
                audioTemp.delete();
            }
        }
    }

    /**
     * 纯文本聊天（兼容现有接口）
     */
    @PostMapping(path = "/text", consumes = MediaType.APPLICATION_JSON_VALUE)
    public MultimodalChatResponse chatText(@RequestBody MultimodalChatRequest request) {
        String answer = chatService.chat(
                request.query(),
                request.instruction(),
                request.conversationId(),
                null,
                null,
                null
        );
        return new MultimodalChatResponse(answer);
    }
}
```

### 5.3 智谱 GLM-ASR-2512 语音转写模型说明

**GLM-ASR-2512** 是智谱于 2025 年 12 月发布的语音转写模型，与 Spring AI 的 `TranscriptionModel` 抽象高度兼容。

| 属性 | 详情 |
|------|------|
| 模型名 | `glm-asr-2512` |
| API 端点 | `POST /v4/audio/transcriptions` |
| 上传限制 | 文件 ≤ 25MB，单段音频 ≤ 30 秒 |
| 支持语言 | 普通话 + 粤语/川话/闽南语/吴语，英语多口音，法/德/日/韩/西班牙语等 |
| 接口风格 | `multipart/form-data`，和 OpenAI 高度一致 |
| 中文准确率 | CER 仅 0.0717，行业领先 |

**音频格式要求**（根据实测脚本）：
- 支持格式：`mp3`, `wav`, `m4a`, `flac`, `ogg`, `webm` 等
- 推荐格式：单声道、16000Hz
- 超长音频（>30s）：需要自动分段处理（可参考 `scripts/glm_asr.py` 中的 `--auto-split` 功能）

**与 Spring AI 集成说明**：
- GLM-ASR 的接口路径是 `/v4/audio/transcriptions`，与 OpenAI 的 `/v1/audio/transcriptions` 形态一致
- 在 Spring AI 中，只需把 `base-url` 指向智谱，`model` 改成 `glm-asr-2512`，可直接使用 `TranscriptionModel` 接入
- 图片理解和语音转写可以都用智谱一个平台搞定

| 视觉模型 API 不可用 | 中 | 视觉模型服务宕机或 API Key 无效 | 主 ChatClient 仍可处理纯文本请求 |
| 大文件上传超时 | 低 | 图片/音频文件较大可能导致请求超时 | 配置 `spring.servlet.multipart` 限制文件大小 |
| 临时文件清理 | 低 | 临时文件可能未及时删除 | 使用 `finally` 块确保清理 |
| 向后兼容性 | 低 | 新增接口不影响现有 `/api/chat` JSON 接口 | 现有接口保持不变 |
| 语音转写成本 | 中 | 语音转写 API 调用成本 | 可选功能，仅在上传音频时触发 |

### 7.2 回滚计划

如需回滚：
1. 保留现有 `ChatController` 和 `AgentService`（不做修改）
2. 禁用或删除新增的 `MultimodalChatController`
3. 配置文件修改为注释状态即可

## 九、注意事项

### 9.1 与现有系统的集成

1. **RagService 对接**：本方案中的 `ExistingRagService` 直接使用 `VectorStore.similaritySearch()` 实现，这是最简化的 RAG 方式。如需复用 `AgentService` 中更完善的 RAG 逻辑（如 `QuestionAnswerAdvisor`），可以自行扩展 `RagService` 接口实现。

2. **会话记忆**：多模态场景下的会话记忆处理策略：
   - 图片通过 `visionChatClient` 先转为文字描述，不直接传 Media 给主 ChatClient
   - 音频通过 `TranscriptionModel` 转写为文本
   - 文字描述和转写文本会进入 RAG 检索和主 ChatClient 对话
   - 推荐方案：只存储转写后的文本，原始媒体文件不进入记忆系统

3. **Skills 集成**：`SpringAiMultimodalChatService` 目前**未集成** `SkillsAdvisor`，这意味着：
   - 技能系统（Skills）的能力**不会**在多模态聊天中生效
   - 如需支持技能系统的多模态调用，需要额外扩展：
     - 将 `SkillsAdvisor` 作为默认 Advisor 添加到 `ChatClient` 中
     - 参考 `AgentService` 中的 `SkillsAdvisor` 使用方式
   - 这是一个有意为之的设计决策，以保持多模态实现的简洁性

### 9.2 未来扩展方向

1. **流式输出**：当前方案返回完整文本，可扩展为 SSE 流式响应
2. **AG-UI 协议**：AgUiController 是否需要支持多模态输入？
3. **Skills 集成**：如需在多模态聊天中启用技能系统
