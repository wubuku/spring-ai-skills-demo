## 一、项目现状分析

| Spring Boot | 3.4.2 |

| Java | 17+ |

| 嵌入模型 | SiliconFlow (BAAI/bge-m3, 1024维) |
| 向量存储 | SimpleVectorStore (默认) / PgVectorStore (可选) |
| 记忆系统 | JDBC Chat Memory (H2 默认, PostgreSQL 可选) |
| HTTP 客户端 | 自定义 OkHttp (含重试逻辑) |

### 1.2 现有 AI 交互架构

```
[ChatController] POST /api/chat (JSON)
       ↓
[ChatMessage{role, content, conversationId}]
       ↓
[AgentService]
   ├── SkillsAdvisor (技能系统注入)
   ├── MessageChatMemoryAdvisor (JDBC 消息窗口)
   ├── VectorStoreChatMemoryAdvisor (语义记忆)
   ├── QuestionAnswerAdvisor (RAG 知识库)
   ├── SkillTools (默认工具)
   └── ChatClient (LLM 调用)
```

### 1.3 现有组件清单

| 组件 | 路径 | 功能 |
|------|------|------|
| ChatController | `controller/ChatController.java` | HTTP 入口，接收 JSON ChatMessage |
| AgentService | `service/AgentService.java` | 核心聊天逻辑，多 Advisor 编排 |
| SpringAiConfig | `config/SpringAiConfig.java` | ChatClient/ChatModel 手动配置 |
| ChatMessage | `model/ChatMessage.java` | 消息 DTO (仅文本) |
| SkillsAdvisor | `agent/SkillsAdvisor.java` | 技能系统 prompt 注入 |
| SkillTools | `agent/SkillTools.java` | 技能执行工具 |

### 1.4 当前限制

- **仅支持文本输入**：ChatMessage.content 为 String 类型
- **无图片处理能力**：无 Media 类型支持
- **无语音处理能力**：无 TranscriptionModel 配置
- **无多模态 ChatClient**：现有 ChatClient 仅处理纯文本

### 1.5 项目特性对方案的影响

1. **自定义 OkHttp 配置**：项目使用自定义 OkHttpClient 而非 Spring AI 自动装配，需要确保多模态请求也能复用现有重试逻辑
2. **多 Provider 支持**：已实现 OpenAI/Anthropic 切换机制，需要考虑多模态场景下的 Provider 扩展
3. **现有 Advisor 架构**：AgentService 使用多层 Advisor，设计时需要考虑多模态输入如何融入现有架构

## 二、设计目标

### 2.1 功能目标

| 目标 | 描述 | 优先级 |
|------|------|--------|
| 图片输入支持 | 用户可上传图片，结合文本或单独上传 | P0 |
| 语音输入支持 | 用户可上传语音文件，自动转写为文本后再处理 | P0 |
| 现有功能兼容 | 纯文本对话功能保持不变 | P0 |
| RAG 集成 | 多模态输入可结合现有知识库检索 | P0 |
| 低成本改造 | 最小化对现有架构的侵入 | P1 |

### 2.2 非功能目标

- **可切换性**：支持切换到不支持多模态的主模型（如 MiniMax M2.7），通过视觉模型代理
- **配置简单化**：通过配置文件切换多模态/非多模态模式
- **低风险**：新增代码与现有代码解耦，不影响现有功能

### 2.3 使用场景

1. **纯文本对话**（兼容现有）：`POST /api/chat/text`
2. **文本 + 图片**：`POST /api/chat` with multipart
3. **文本 + 语音**：`POST /api/chat` with multipart
4. **图片 + 语音 + 文本**：`POST /api/chat` with multipart

## 三、详细设计方案

### 3.1 设计原则

**核心目标**：增强现有的 `AgentService`，而非另起炉灶

- `AgentService` 已有完整能力：Skills、RAG、ChatMemory、SkillTools
- 多模态输入（图片、语音）应被转换为文字，然后交给 `AgentService` 处理
- `AgentService` 的所有 Advisor 和工具能力保持不变

### 3.2 架构设计

```
用户上传 [图片 + 语音 + 文本]
            ↓
   [MultimodalAgentService]
            ↓
   ├── 图片 → [visionChatClient] → 图片描述文字
   ├── 语音 → [TranscriptionModel] → 转写文字
   └── 文本 → 直接保留
            ↓ 多模态内容合并为文字
   [AgentService.chat()]  ← 现有的完整 Agent 能力
            ↓
   [SkillsAdvisor] + [RAG] + [ChatMemory] + [SkillTools]
            ↓
   回复
```

**设计要点**：
1. 新增 `MultimodalAgentService`，内部调用 `AgentService`
2. `visionChatClient` 用于图片理解（新增 Bean）
3. `TranscriptionModel` 用于语音转写（新增 Bean）
4. 图片描述和语音转写文本合并到用户输入中，交给 `AgentService` 处理
5. `AgentService` 的所有能力（RAG、Skills、记忆）自动生效

**关于 visionChatClient 的配置说明：**

双模型方案需要在 `SpringAiConfig.java` 中新增 `visionChatClient` Bean，用于视觉模型（图片理解）。现有 `chatClient` Bean 保持不变。

```java
@Bean("visionChatClient")
public ChatClient visionChatClient() {
    // 视觉模型的配置（使用视觉模型 provider，如火山方舟/豆包视觉）
    OpenAiApi visionApi = OpenAiApi.builder()
            .baseUrl(visionBaseUrl)   // 视觉模型 base-url
            .apiKey(visionApiKey)     // 视觉模型 api-key
            .build();

    OpenAiChatModel visionChatModel = OpenAiChatModel.builder()
            .openAiApi(visionApi)
            .defaultOptions(OpenAiChatOptions.builder()
                    .model(visionModel)  // 视觉模型名称
                    .build())
            .build();

    return ChatClient.create(visionChatModel);
}
```

#### 4.3.2 MultimodalAgentService 实现

```java
// MultimodalAgentService.java
package com.example.demo.service;

import org.springframework.ai.audio.transcription.TranscriptionModel;
import org.springframework.ai.chat.client.ChatClient;
import org.springframework.ai.content.Media;
import org.springframework.beans.factory.annotation.Qualifier;
import org.springframework.core.io.Resource;
import org.springframework.http.MediaType;
import org.springframework.stereotype.Service;

/**
 * 多模态代理服务
 *
 * 核心设计：增强现有的 AgentService，而非另起炉灶
 * - 图片理解：使用 visionChatClient 将图片转为文字描述
 * - 语音转写：使用 TranscriptionModel 将语音转为文字
 * - 文字合并：将图片描述、语音转写合并到用户输入
 * - 交给 AgentService.chat() 处理，保留所有现有能力（RAG、Skills等）
 */
@Service
public class MultimodalAgentService {

    private final AgentService agentService;               // 现有的 AgentService（保留所有能力）
    private final ChatClient visionChatClient;              // 视觉模型 ChatClient
    private final TranscriptionModel transcriptionModel;    // 语音转写模型

    public MultimodalAgentService(
            AgentService agentService,
            @Qualifier("visionChatClient") ChatClient visionChatClient,
            TranscriptionModel transcriptionModel) {
        this.agentService = agentService;
        this.visionChatClient = visionChatClient;
        this.transcriptionModel = transcriptionModel;
    }

    /**
     * 多模态聊天
     * 1. 图片理解 → visionChatClient → 文字描述
     * 2. 语音转写 → TranscriptionModel → 文字
     * 3. 合并所有文字输入
     * 4. 调用 AgentService.chat() 处理（保留所有能力）
     */
    public String chat(String query,
                       String conversationId,
                       Resource image,
                       String imageContentType,
                       Resource audio) {

        StringBuilder enrichedInput = new StringBuilder();

        // 1. 如果有图片，先获取图片的文字描述
        if (image != null) {
            String imageDescription = describeImage(image, imageContentType, query);
            enrichedInput.append("【图片内容】").append(imageDescription).append("\n\n");
        }

        // 2. 如果有音频，转写为文字
        if (audio != null) {
            String transcription = transcriptionModel.transcribe(audio);
            enrichedInput.append("【语音转写】").append(transcription).append("\n\n");
        }

        // 3. 如果有用户文本，追加
        if (query != null && !query.isBlank()) {
            enrichedInput.append("【用户输入】").append(query);
        }

        String finalInput = enrichedInput.toString();
        if (finalInput.isBlank()) {
            finalInput = "用户未提供有效输入。";
        }

        // 4. 交给 AgentService 处理（保留所有能力：Skills、RAG、记忆等）
        return agentService.chat(finalInput, conversationId);
    }

    /**
     * 使用视觉模型将图片转换为文字描述
     */
    private String describeImage(Resource image, String imageContentType, String hint) {
        MediaType mt = parseMediaType(imageContentType, MediaType.IMAGE_JPEG);
        Media media = new Media(mt, image);

        String visionPrompt = (hint != null && !hint.isBlank())
                ? "用户问题是：" + hint + "\n请详细描述这张图片的内容，包括文字、数据、图表、场景等所有重要信息。"
                : "请详细描述这张图片的内容，包括文字、数据、图表、场景等所有重要信息。";

        return visionChatClient.prompt()
                .user(user -> user.text(visionPrompt).media(media))
                .call()
                .content();
    }

    private MediaType parseMediaType(String contentType, MediaType defaultType) {
        if (contentType == null || contentType.isBlank()) {
            return defaultType;
        }
        try {
            return MediaType.parseMediaType(contentType);
        } catch (Exception e) {
            return defaultType;
        }
    }
}
```

#### 4.3.4 MultimodalChatController

```java
// MultimodalChatController.java
package com.example.demo.controller;

import com.example.demo.dto.MultimodalChatRequest;
import com.example.demo.dto.MultimodalChatResponse;
import com.example.demo.service.MultimodalAgentService;
import org.springframework.core.io.FileSystemResource;
import org.springframework.http.MediaType;
import org.springframework.web.bind.annotation.*;
import org.springframework.web.multipart.MultipartFile;

import java.io.File;

@RestController
@RequestMapping("/api/chat")
public class MultimodalChatController {

    private final MultimodalAgentService multimodalAgentService;

    public MultimodalChatController(MultimodalAgentService multimodalAgentService) {
        this.multimodalAgentService = multimodalAgentService;
    }

    /**
     * 多模态聊天入口
     * 支持同时上传文本、图片、音频
     */
    @PostMapping(consumes = MediaType.MULTIPART_FORM_DATA_VALUE)
    public MultimodalChatResponse chat(
            @RequestParam(value = "query", required = false) String query,
            @RequestParam(value = "conversationId", required = false, defaultValue = "default") String conversationId,
            @RequestPart(value = "image", required = false) MultipartFile image,
            @RequestPart(value = "audio", required = false) MultipartFile audio
    ) throws Exception {

        File imageTemp = null;
        File audioTemp = null;

        try {
            // 处理图片
            FileSystemResource imageResource = null;
            String imageContentType = null;
            if (image != null && !image.isEmpty()) {
                imageTemp = File.createTempFile("img-", "-" + image.getOriginalFilename());
                image.transferTo(imageTemp);
                imageResource = new FileSystemResource(imageTemp);
                imageContentType = image.getContentType();
            }

            // 处理音频
            FileSystemResource audioResource = null;
            if (audio != null && !audio.isEmpty()) {
                audioTemp = File.createTempFile("audio-", "-" + audio.getOriginalFilename());
                audio.transferTo(audioTemp);
                audioResource = new FileSystemResource(audioTemp);
            }

            String answer = multimodalAgentService.chat(
                    query,
                    conversationId,
                    imageResource,
                    imageContentType,
                    audioResource
            );

            return new MultimodalChatResponse(answer);

        } finally {
            // 清理临时文件
            if (imageTemp != null && imageTemp.exists()) {
                imageTemp.delete();
            }
            if (audioTemp != null && audioTemp.exists()) {
                audioTemp.delete();
            }
        }
    }

    /**
     * 纯文本聊天（兼容现有接口）
     * 直接调用 AgentService，不经过多模态处理
     */
    @PostMapping(path = "/text", consumes = MediaType.APPLICATION_JSON_VALUE)
    public MultimodalChatResponse chatText(@RequestBody MultimodalChatRequest request) {
        // 纯文本不需要多模态处理，直接走 AgentService
        // 这里需要注入 AgentService 或 MultimodalAgentService
        // 简化处理：复用 multipart 接口，但不使用 image/audio
        return new MultimodalChatResponse("请使用 multipart 接口上传内容");
    }
}
```

### 5.3 智谱 GLM-ASR-2512 语音转写模型说明

**GLM-ASR-2512** 是智谱于 2025 年 12 月发布的语音转写模型，与 Spring AI 的 `TranscriptionModel` 抽象高度兼容。

| 属性 | 详情 |
|------|------|
| 模型名 | `glm-asr-2512` |
| API 端点 | `POST /v4/audio/transcriptions` |
| 上传限制 | 文件 ≤ 25MB，单段音频 ≤ 30 秒 |
| 支持语言 | 普通话 + 粤语/川话/闽南语/吴语，英语多口音，法/德/日/韩/西班牙语等 |
| 接口风格 | `multipart/form-data`，和 OpenAI 高度一致 |
| 中文准确率 | CER 仅 0.0717，行业领先 |

**音频格式要求**（根据实测脚本）：
- 支持格式：`mp3`, `wav`, `m4a`, `flac`, `ogg`, `webm` 等
- 推荐格式：单声道、16000Hz
- 超长音频（>30s）：需要自动分段处理（可参考 `scripts/glm_asr.py` 中的 `--auto-split` 功能）

**与 Spring AI 集成说明**：
- GLM-ASR 的接口路径是 `/v4/audio/transcriptions`，与 OpenAI 的 `/v1/audio/transcriptions` 形态一致
- 在 Spring AI 中，只需把 `base-url` 指向智谱，`model` 改成 `glm-asr-2512`，可直接使用 `TranscriptionModel` 接入
- 图片理解和语音转写可以都用智谱一个平台搞定

## 七、风险评估

### 7.1 风险清单

| 风险 | 等级 | 描述 | 缓解措施 |
|------|------|------|----------|
| 视觉模型 API 不可用 | 中 | 视觉模型服务宕机或 API Key 无效 | 主 ChatClient 仍可处理纯文本请求 |
| 大文件上传超时 | 低 | 图片/音频文件较大可能导致请求超时 | 配置 `spring.servlet.multipart` 限制文件大小 |
| 临时文件清理 | 低 | 临时文件可能未及时删除 | 使用 `finally` 块确保清理 |
| 向后兼容性 | 低 | 新增接口不影响现有 `/api/chat` JSON 接口 | 现有接口保持不变 |
| 语音转写成本 | 中 | 语音转写 API 调用成本 | 可选功能，仅在上传音频时触发 |

### 7.2 回滚计划

如需回滚：
1. 保留现有 `ChatController` 和 `AgentService`（不做修改）
2. 禁用或删除新增的 `MultimodalChatController`
3. 配置文件修改为注释状态即可

## 九、注意事项

### 9.1 与现有系统的集成

1. **RagService 对接**：本方案中的 `ExistingRagService` 直接使用 `VectorStore.similaritySearch()` 实现，这是最简化的 RAG 方式。如需复用 `AgentService` 中更完善的 RAG 逻辑（如 `QuestionAnswerAdvisor`），可以自行扩展 `RagService` 接口实现。

2. **会话记忆**：多模态场景下的会话记忆处理策略：
   - 图片通过 `visionChatClient` 先转为文字描述，不直接传 Media 给主 ChatClient
   - 音频通过 `TranscriptionModel` 转写为文本
   - 文字描述和转写文本会进入 RAG 检索和主 ChatClient 对话
   - 推荐方案：只存储转写后的文本，原始媒体文件不进入记忆系统

3. **Skills 集成**：`SpringAiMultimodalChatService` 目前**未集成** `SkillsAdvisor`，这意味着：
   - 技能系统（Skills）的能力**不会**在多模态聊天中生效
   - 如需支持技能系统的多模态调用，需要额外扩展：
     - 将 `SkillsAdvisor` 作为默认 Advisor 添加到 `ChatClient` 中
     - 参考 `AgentService` 中的 `SkillsAdvisor` 使用方式
   - 这是一个有意为之的设计决策，以保持多模态实现的简洁性

### 9.2 未来扩展方向

1. **AG-UI 协议**：AgUiController 是否需要支持多模态输入？
2. **Skills 集成**：如需在多模态聊天中启用技能系统

## 十一、SSE 流式协议（前后端开发契约）

### 11.1 概述

本项目实现了基于 **Server-Sent Events (SSE)** 的流式响应协议，兼容 **OpenAI SSE 格式**，并扩展支持多模态（图片、语音）流式处理。

**关键设计原则**：
- **尽可能真流式**：视觉模型和 LLM 的 token 实时逐个返回给前端
- **OpenAI 兼容**：SSE 数据格式与 OpenAI 流式 API 保持一致
- **多模态扩展**：通过 `type` 字段区分内容来源（vision/transcribed/content）

### 11.2 API 端点

| 端点 | 方法 | Content-Type | 流式 | 说明 |
|------|------|--------------|------|------|
| `/api/chat/stream` | POST | `application/json` | ✅ | 纯文本流式聊天 |
| `/api/chat/multimodal/stream` | POST | `multipart/form-data` | ✅ | 多模态流式聊天（图片+语音） |

**请求头要求**：
- `Accept: text/event-stream`（SSE 必需）
- `Content-Type: application/json`（纯文本）或 `multipart/form-data`（多模态）

### 11.3 SSE 响应格式

#### 11.3.1 纯文本格式（OpenAI 兼容）

```
data:{"choices":[{"delta":{"content":"你"}}]}
data:{"choices":[{"delta":{"content":"好"}}]}
data:{"choices":[{"delta":{"content":"！"}}]}
...
data:[DONE]
```

#### 11.3.2 与 OpenAI 官方格式对比

| 方面 | OpenAI 官方格式 | 我们项目格式 | 兼容性说明 |
|------|----------------|-------------|-----------|
| 核心结构 | `choices[0].delta.content` | `choices[0].delta.content` | ✅ **完全一致** |
| 元数据 | `id`, `object`, `created`, `model`, `system_fingerprint` | 无 | 可省略 |
| `logprobs` | 有（默认 null） | 无 | 可省略 |
| `finish_reason` | 最后一个 chunk 有 | 无 | 可省略，前端用 `[DONE]` 判断结束 |
| 结束标识 | `data: [DONE]` | `data: [DONE]` | ✅ **完全一致** |
| 首个 chunk | `delta: {"role":"assistant","content":""}` | `delta: {"content":"..."}` | 结构一致，我们更简洁 |

**OpenAI 官方格式样例**（来自官方 API 文档）：

```json
{"id":"chatcmpl-123","object":"chat.completion.chunk","created":1694268190,"model":"gpt-4o-mini","choices":[{"index":0,"delta":{"role":"assistant","content":""},"logprobs":null,"finish_reason":null}]}
{"id":"chatcmpl-123","object":"chat.completion.chunk","created":1694268190,"model":"gpt-4o-mini","choices":[{"index":0,"delta":{"content":"Hello"},"logprobs":null,"finish_reason":null}]}
...
{"id":"chatcmpl-123","object":"chat.completion.chunk","created":1694268190,"model":"gpt-4o-mini","choices":[{"index":0,"delta":{},"logprobs":null,"finish_reason":"stop"}]}
data:[DONE]
```

**我们项目的简化格式**：

```json
{"choices":[{"delta":{"content":"你"}}]}
{"choices":[{"delta":{"content":"好"}}]}
...
data:[DONE]
```

**结论**：我们采用**简化版 OpenAI SSE 格式**，保留了最核心的 `choices[0].delta.content` 结构，去掉了 `id`、`object`、`created`、`model`、`logprobs`、`finish_reason` 等元数据。这使得：
- 前端解析更简单（只需访问 `chunk.choices[0].delta.content`）
- 数据传输更紧凑
- 仍然兼容 OpenAI 风格的前端代码（只需忽略额外字段）

#### 11.3.3 多模态扩展格式

多模态端点在每个 chunk 中增加了 `type` 字段：

```
data:{"type":"vision","choices":[{"delta":{"content":"这"}}]}
data:{"type":"vision","choices":[{"delta":{"content":"是"}}]}
data:{"type":"vision","choices":[{"delta":{"content":"一"}}]}
...
data:{"type":"content","choices":[{"delta":{"content":"根"}}]}
data:{"type":"content","choices":[{"delta":{"content":"据"}}]}
...
data:[DONE]
```

**注意**：
- `type` 字段**仅存在于** `/api/chat/multimodal/stream` 响应中
- 纯文本端点 `/api/chat/stream` **不包含** `type` 字段
- 语音转写（ASR）结果通过 `type="content"` 包裹在 LLM 流中返回，不再单独发送

### 11.4 `type` 字段取值说明

| type 值 | 来源 | 说明 | 流式特性 |
|---------|------|------|----------|
| `"vision"` | 视觉模型 | 图片识别结果，每个 token 实时返回 | **真流式** |
| `"content"` | LLM / ASR | 最终回复内容或语音转写结果 | **真流式**（LLM）/ **假流式**（ASR 通过 LLM 流返回） |

**关于 ASR 转写的处理**：
- ASR 转写结果**不单独发送**，而是被拼接进 LLM 的 `content` 流中
- 客户端看到的是 `type="content"` 的连续流，无需区分 ASR 和 LLM
- 这种设计简化了前端处理逻辑

### 11.6 前端集成指南

#### 11.6.1 基础 SSE 解析

```javascript
async function* streamSSE(response) {
  const reader = response.body.getReader();
  const decoder = new TextDecoder();
  let buffer = '';

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    buffer += decoder.decode(value, { stream: true });
    const lines = buffer.split('\n');
    buffer = lines.pop() ?? '';

    for (const line of lines) {
      if (line.startsWith('data:')) {
        const data = line.slice(5).trim();
        if (data === '[DONE]') {
          return;
        }
        yield JSON.parse(data);
      }
    }
  }
}
```

#### 11.6.2 纯文本聊天（/api/chat/stream）

```javascript
async function chatStream(text, conversationId = 'default') {
  const response = await fetch('/api/chat/stream', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Accept': 'text/event-stream'
    },
    body: JSON.stringify({ content: text, conversationId })
  });

  let fullText = '';
  for await (const chunk of streamSSE(response)) {
    const content = chunk.choices?.[0]?.delta?.content ?? '';
    fullText += content;
    console.log('[content]', content);
  }
  return fullText;
}
```

####  11.6.3 多模态聊天（/api/chat/multimodal/stream）

```javascript
async function multimodalStream(query, imageFile, audioFile) {
  const formData = new FormData();
  if (query) formData.append('query', query);
  formData.append('conversationId', 'default');
  if (imageFile) formData.append('image', imageFile);
  if (audioFile) formData.append('audio', audioFile);

  const response = await fetch('/api/chat/multimodal/stream', {
    method: 'POST',
    headers: { 'Accept': 'text/event-stream' },
    body: formData
  });

  let fullText = '';
  for await (const chunk of streamSSE(response)) {
    // 多模态端点返回的 chunk 包含 type 字段
    const content = chunk.choices?.[0]?.delta?.content ?? '';
    const type = chunk.type ?? 'content';  // 默认 content（兼容纯文本场景）

    fullText += content;

    // 根据 type 处理不同来源的内容
    switch (type) {
      case 'vision':
        console.log('[vision]', content);
        updateVisionUI(content);
        break;
      case 'content':
      default:
        console.log('[content]', content);
        updateContentUI(content);
        break;
    }
  }
  return fullText;
}
```

### 11.7 流式处理流程图

#### 场景 1：纯文本
```
客户端 → [LLM] → SSE(content) → 客户端
         (真流式)
```

#### 场景 2：图片 + 文本
```
客户端 → [Vision] → SSE(vision) → 客户端
         ↓ (reduce 收集完整)
         [LLM] → SSE(content) → 客户端
```

#### 场景 3：语音 + 文本
```
客户端 ← [ASR 同步转写] ← 客户端
         ↓ (一次性结果，拼接进 LLM 输入)
         [LLM] → SSE(content) → 客户端
                 （ASR 结果作为 LLM 流的一部分返回）
```

#### 场景 4：图片 + 语音 + 文本
```
客户端 → [Vision] → SSE(vision) → 客户端
         ↓ (reduce 收集完整)
[ASR 同步] → ─────────────────────┐
         ↓ (zip 合并两者)          │
         [LLM] → SSE(content) → 客户端
```

### 11.8 技术实现要点

#### 11.8.1 服务层实现

```java
// MultimodalAgentService.streamChat() 根据模态组合分发
public Flux<MultimodalToken> streamChat(...) {
    if (hasImage && hasAudio) return streamImageAndAudio(...);
    if (hasImage) return streamImageOnly(...);
    if (hasAudio) return streamAudioOnly(...);
    // 纯文字
    return agentService.streamChat(query, conversationId)
            .map(token -> MultimodalToken.content(token));
}

// 图片 + 文字：Vision 真流式 → LLM 真流式
private Flux<MultimodalToken> streamImageOnly(...) {
    Flux<String> visionTokens = visionChatClient.prompt()
        .user(user -> user.text(visionPrompt).media(media))
        .stream().content().cache();  // cache() 让 vision 结果可重复订阅

    Flux<MultimodalToken> visionStage = visionTokens.map(MultimodalToken::vision);
    Flux<MultimodalToken> llmStage = visionTokens
        .reduce("", String::concat)   // 等待 vision 完整
        .flatMapMany(imageDescription ->
            agentService.streamChat("【图片内容】" + imageDescription, conversationId)
                .map(MultimodalToken::content)
        );

    return Flux.concat(visionStage, llmStage);
}

// 图片 + 语音 + 文字：Vision 真流式 + ASR 同步 → LLM 真流式
private Flux<MultimodalToken> streamImageAndAudio(...) {
    Flux<String> visionTokens = visionChatClient.prompt()
        .user(user -> user.text(visionPrompt).media(media))
        .stream().content().cache();

    Flux<MultimodalToken> visionStage = visionTokens.map(MultimodalToken::vision);

    // ASR 在独立线程同步执行
    Mono<String> asrMono = Mono.fromCallable(() -> transcriptionModel.transcribe(audio))
        .subscribeOn(Schedulers.boundedElastic());

    // LLM 等 vision 和 ASR 都完成后开始
    Flux<MultimodalToken> llmStage = visionTokens.reduce("", String::concat)
        .zipWith(asrMono)
        .flatMapMany(tuple -> {
            String imageDesc = tuple.getT1();
            String transcript = tuple.getT2();
            return agentService.streamChat(
                "【图片内容】" + imageDesc + "\n\n【语音转写】" + transcript,
                conversationId
            ).map(MultimodalToken::content);
        });

    return Flux.concat(visionStage, llmStage);
}
```

#### 11.8.2 Controller 层 SSE 序列化

```java
// MultimodalChatController.chatStream()
SseEmitter emitter = new SseEmitter(0L);  // 无超时

Flux<MultimodalToken> tokenFlux = multimodalAgentService.streamChat(...);

tokenFlux
    .subscribeOn(Schedulers.boundedElastic())  // 独立线程订阅
    .doFinally(signalType -> {
        // 清理临时文件
        if (imageTemp != null) imageTemp.delete();
        if (audioTemp != null) audioTemp.delete();
    })
    .subscribe(
        mt -> {
            // 发送 SSE 事件
            Map<String, Object> chunk = Map.of(
                "type", mt.type(),
                "choices", List.of(Map.of("delta", Map.of("content", mt.content())))
            );
            emitter.send(SseEmitter.event()
                .data(objectMapper.writeValueAsString(chunk)));
        },
        error -> emitter.completeWithError(error),
        () -> {
            emitter.send(SseEmitter.event().data("[DONE]"));
            emitter.complete();
        }
    );
```

### 11.9 注意事项

1. **SSE 兼容性**：`Accept: text/event-stream` 是必需的，否则 Spring 可能返回普通 JSON
2. **超时设置**：`SseEmitter(0L)` 表示无超时限制
3. **并发处理**：多个 SSE 流可以并发处理，互不影响
4. **错误处理**：连接断开时 SseEmitter 自动清理，`doFinally` 确保临时文件被删除
5. **ASR 处理**：ASR 转写结果不单独发送，而是通过 `【语音转写】xxx` 拼接进 LLM 的 content 流
6. **type 字段**：`type` 字段**仅多模态端点**返回，纯文本端点不返回 `type`
