# 支持多模态输入（图片、语音）

## 用户提问

> 我们已经基于 Spring AI 开发一个典型的 RAG 系统。使用 ChatClient 实现了“用户输入自然语言、我们基于知识库生成回复”的基础功能。现在希望支持“用户上传一张图片”或者“用户说一段话（上传语音数据）”，系统也可以给出合理的“回复”。回复就是文本的就好。
>
> 目前不需要实时语音对话，也不需要“回复图片”。语音部分只需要采用语音转写为文本后，再调用 LLM 模型即可。
>
> 背景清楚了，请你给出一个完整、完善的、基于 Spring AI 的 Demo 应用。

下面是一个“可直接跑起来”的 Spring Boot Demo，基于 Spring AI，支持三种输入：

- 只输入文本（你现有的 RAG 场景）。  
- 文本 + 图片（多模态视觉理解）。  
- 文本 + 语音文件（先转写为文本，再走 RAG + LLM）。  

设计上：  
- 图片通过 `ChatClient` 的多模态输入能力，使用 `Media` 抽象传入。 [docs.spring](https://docs.spring.io/spring-ai/reference/api/chatclient.html)
- 音频通过 `TranscriptionModel` 抽象转写为文本，再拼进同一套对话流程。 [docs.spring](https://docs.spring.io/spring-ai/reference/api/audio/transcriptions.html)
- 和你现有 RAG 系统集成，只需要在一个 `RagService` 里调用你现有的向量检索逻辑即可。

***

## 设计目标与总体结构

目标：

- 保持你现有的“文本 RAG 对话”代码结构不大动，只是把“输入”扩展为多模态。  
- 对上暴露统一接口：`/api/chat`，前端可以同时/分别上传文本、图片、音频。  
- 对下尽量只依赖 Spring AI 抽象：`ChatClient` + `TranscriptionModel`。 [docs.spring](https://docs.spring.io/spring-ai/reference/api/chatclient.html)

主要类：

- `DemoApplication`：启动类  
- `AiConfig`：可选配置类，注入 `ChatClient`  
- `RagService`：抽象你的知识库检索逻辑  
- `MultimodalChatService`：核心业务，多模态 + RAG + LLM  
- `ChatController`：HTTP 接口（文本/图片/音频）  
- DTO：`ChatRequest` / `ChatResponse`

```yaml

  port: 8080

spring:
  servlet:
    multipart:
      max-file-size: 20MB
      max-request-size: 20MB

  ai:
    openai:
      # 如果直连 OpenAI：
      # base-url: https://api.openai.com/v1

      base-url: ${OPENAI_BASE_URL:https://api.openai.com/v1}
      api-key: ${OPENAI_API_KEY}

      chat:
        options:
          # 支持多模态的模型，例如 gpt-4o / gpt-4o-mini 或国产兼容模型名
          model: ${CHAT_MODEL:gpt-4o-mini}

      audio:
        transcription:
          # 若用同一平台，一般可共用 base-url / api-key
          base-url: ${OPENAI_BASE_URL:https://api.openai.com/v1}
          api-key: ${OPENAI_API_KEY}
          options:
            model: ${ASR_MODEL:whisper-1}
            response-format: json

    model:
      audio:
        transcription: openai
```

要接国内平台（阿里百炼 / 智谱 / 千帆）的 OpenAI 兼容接口，只需要把 `OPENAI_BASE_URL` / `OPENAI_API_KEY` / 模型名换掉即可，Spring AI 本身不需要改代码。 [ppio](https://ppio.com/docs/llms-full.txt?from=ppinfra)

## 多模态 + RAG + LLM 业务 Service

核心逻辑：

1. 如果有音频：用 `TranscriptionModel.transcribe(Resource)` 得到文本。 [docs.spring](https://docs.spring.io/spring-ai/docs/1.1.x/api/org/springframework/ai/audio/transcription/TranscriptionModel.html)
2. 把「原始 query + 语音转写文本」拼接成最终 userText。  
3. 调 `RagService.buildContext(userText)` 拿到知识库上下文。  
4. 如果有图片：通过 `ChatClient` 的 `user().media(Media)` 把图片作为多模态输入。 [docs.spring](https://docs.spring.io/spring-ai/docs/current/api/org/springframework/ai/content/class-use/Media.html)
5. 用 ChatClient 统一发起 LLM 调用，返回文本答案。

```java
// MultimodalChatService.java
package com.example.demo.service;

import org.springframework.core.io.Resource;

public interface MultimodalChatService {

    String chat(String query,
                String instruction,
                Resource image,
                String imageContentType,
                Resource audio);
}
```

```java
// SpringAiMultimodalChatService.java
package com.example.demo.service.impl;

import com.example.demo.service.MultimodalChatService;
import com.example.demo.service.RagService;
import org.springframework.ai.audio.transcription.TranscriptionModel;
import org.springframework.ai.chat.client.ChatClient;
import org.springframework.ai.content.Media;
import org.springframework.core.io.Resource;
import org.springframework.http.MediaType;
import org.springframework.stereotype.Service;

@Service
public class SpringAiMultimodalChatService implements MultimodalChatService {

    private final ChatClient chatClient;
    private final TranscriptionModel transcriptionModel;
    private final RagService ragService;

    public SpringAiMultimodalChatService(
            ChatClient chatClient,
            TranscriptionModel transcriptionModel,
            RagService ragService
    ) {
        this.chatClient = chatClient;
        this.transcriptionModel = transcriptionModel;
        this.ragService = ragService;
    }

    @Override
    public String chat(String query,
                       String instruction,
                       Resource image,
                       String imageContentType,
                       Resource audio) {

        // 1. 整理用户文本输入（文本 + 语音转写）
        StringBuilder userTextBuilder = new StringBuilder();
        if (query != null && !query.isBlank()) {
            userTextBuilder.append("用户文本: ").append(query).append("\n");
        }

        if (audio != null) {
            String transcription = transcriptionModel.transcribe(audio); // Spring AI 抽象方法[web:123][web:107]
            userTextBuilder.append("语音转写: ").append(transcription).append("\n");
        }

        String userText = userTextBuilder.toString();
        if (userText.isBlank()) {
            userText = "用户没有提供文本，仅提供了图片/语音，请根据其它模态进行理解和回答。";
        }

        // 2. RAG：根据综合后的 userText 做知识检索
        String context = ragService.buildContext(userText);

        // 3. 构造 system / user 消息
        String systemPrompt = """
                你是一个知识库问答助手。请严格基于提供的知识上下文回答用户问题。
                如果知识库中没有足够信息，请坦诚说明，并给出你基于常识的合理推断。
                """;

        String finalUserText = """
                [知识上下文]
                %s

                [用户输入及转写]
                %s
                """.formatted(context, userText);

        // 4. 构造 ChatClient 调用（可带图片，也可不带）
        ChatClient.PromptUserSpec userSpec = user -> user.text(finalUserText);

        if (image != null) {
            MediaType mt = MediaType.IMAGE_JPEG;
            if (imageContentType != null && !imageContentType.isBlank()) {
                try {
                    mt = MediaType.parseMediaType(imageContentType);
                } catch (Exception ignored) {
                }
            }
            Media media = new Media(mt, image); // Spring AI 多模态 Media 抽象[web:104]

            userSpec = user -> user.text(finalUserText)
                                   .media(media); // 把图片作为多模态输入之一[web:102][web:104]
        }

        return chatClient.prompt()
                .system(systemPrompt)
                .user(userSpec)
                .call()
                .content(); // 返回文本答案[web:102]
    }
}
```

## Controller：统一多模态入口

接口设计为：`POST /api/chat`，支持同时/分别上传：

- `query`：文本  
- `instruction`：可选，额外指令  
- `image`：图片  
- `audio`：音频  

```java
// ChatController.java
package com.example.demo.web;

import com.example.demo.dto.ChatRequest;
import com.example.demo.dto.ChatResponse;
import com.example.demo.service.MultimodalChatService;
import org.springframework.core.io.FileSystemResource;
import org.springframework.http.MediaType;
import org.springframework.web.bind.annotation.*;
import org.springframework.web.multipart.MultipartFile;

import java.io.File;

@RestController
@RequestMapping("/api/chat")
public class ChatController {

    private final MultimodalChatService chatService;

    public ChatController(MultimodalChatService chatService) {
        this.chatService = chatService;
    }

    @PostMapping(consumes = MediaType.MULTIPART_FORM_DATA_VALUE)
    public ChatResponse chat(
            @RequestParam(value = "query", required = false) String query,
            @RequestParam(value = "instruction", required = false) String instruction,
            @RequestPart(value = "image", required = false) MultipartFile image,
            @RequestPart(value = "audio", required = false) MultipartFile audio
    ) throws Exception {

        File imageTemp = null;
        File audioTemp = null;

        try {
            FileSystemResource imageResource = null;
            String imageContentType = null;
            if (image != null && !image.isEmpty()) {
                imageTemp = File.createTempFile("img-", "-" + image.getOriginalFilename());
                image.transferTo(imageTemp);
                imageResource = new FileSystemResource(imageTemp);
                imageContentType = image.getContentType();
            }

            FileSystemResource audioResource = null;
            if (audio != null && !audio.isEmpty()) {
                audioTemp = File.createTempFile("audio-", "-" + audio.getOriginalFilename());
                audio.transferTo(audioTemp);
                audioResource = new FileSystemResource(audioTemp);
            }

            String answer = chatService.chat(
                    query,
                    instruction,
                    imageResource,
                    imageContentType,
                    audioResource
            );

            return new ChatResponse(answer);

        } finally {
            if (imageTemp != null && imageTemp.exists()) {
                // 简单清理临时文件
                imageTemp.delete();
            }
            if (audioTemp != null && audioTemp.exists()) {
                audioTemp.delete();
            }
        }
    }

    // 如果你需要纯 JSON 文本接口（兼容你现有前端），可以再额外提供一个 text-only 版本：
    @PostMapping(path = "/text", consumes = MediaType.APPLICATION_JSON_VALUE)
    public ChatResponse chatText(@RequestBody ChatRequest request) {
        String answer = chatService.chat(
                request.query(),
                request.instruction(),
                null,
                null,
                null
        );
        return new ChatResponse(answer);
    }
}
```

## 使用方式示例

### 1）纯文本（兼容你现有 RAG 前端）

```bash
curl -X POST "http://localhost:8080/api/chat/text" \
  -H "Content-Type: application/json" \
  -d '{
        "query": "请基于知识库介绍一下你们的计费规则",
        "instruction": "回答请简洁，限制在 200 字内"
      }'
```

### 2）文本 + 图片

```bash
curl -X POST "http://localhost:8080/api/chat" \
  -F "query=这张截图里的错误是什么意思？请结合知识库帮我排查。" \
  -F "image=@/path/to/screenshot.png"
```

后端会把图片作为 `Media` 多模态输入传给支持视觉的 LLM 模型，再结合 RAG 上下文生成文本回复。 [docs.spring](https://docs.spring.io/spring-ai/docs/current/api/org/springframework/ai/content/class-use/Media.html)

### 3）语音 + 文本

```bash
curl -X POST "http://localhost:8080/api/chat" \
  -F "query=补充一下，我刚才语音里提到的问题是关于退款流程的。" \
  -F "audio=@/path/to/voice.m4a"
```

后端会先通过 `TranscriptionModel` 把语音转写为文本，再把“原始 query + 转写文本”一起送入 RAG + LLM。 [docs.spring](https://docs.spring.io/spring-ai/reference/api/audio/transcriptions.html)

## 和你现在项目的对接建议

- 你现有的“文本 RAG 流程”，只要实现到 `RagService.buildContext(String userQuery)` 里即可，不需要大改其他逻辑。

好，这是一个非常典型的"双模型协作"架构。让我先说清楚改法的核心思路，再给出具体代码。

## 核心架构变化

原来是"一个支持多模态的 ChatClient 包办一切"：

```
用户 [文本 + 图片] → 单一 ChatClient(多模态模型) → 回复
```

改后是"视觉模型先描述图片，主模型再综合回答"：

```
用户 [图片]  → visionChatClient(豆包视觉) → 图片描述文字
用户 [音频]  → TranscriptionModel(GLM-ASR)  → 转写文字
用户 [文本]  → 直接保留
                    ↓ 三路文字合并
              RAG 检索 + mainChatClient(MiniMax M2.7) → 回复
```

字节跳动的视觉模型通过**火山方舟（ARK）平台**提供，模型名 `doubao-vision-pro-32k`，且支持 OpenAI 兼容接口。 [volcengine](https://www.volcengine.com/docs/82379/1362931)

***

## 第一步：application.yml

原本只有一套 `spring.ai.openai.*`，现在需要支持两个不同平台，改成**只用 properties 存配置，不再依赖 Spring AI 自动装配 ChatClient**：

```yaml
server:
  port: 8080

spring:
  servlet:
    multipart:
      max-file-size: 20MB
      max-request-size: 20MB

# 不再使用 spring.ai.openai.* 自动装配 ChatClient

      model: MiniMax-M2.7

    # 视觉模型：字节跳动豆包视觉，通过火山方舟平台
    vision:
      base-url: https://ark.cn-beijing.volces.com/api/v3
      api-key: ${ARK_API_KEY}
      model: doubao-vision-pro-32k   # 或你在火山方舟控制台创建的 endpoint ID

    # 语音转写：智谱 GLM-ASR（接口兼容 OpenAI transcriptions）
    transcription:
      base-url: https://open.bigmodel.cn/api/paas
      api-key: ${ZHIPU_API_KEY}
      model: glm-asr-2512
```

## 第二步：AiConfig.java（最关键的改动）

Spring AI 允许手动通过 `OpenAiApi` + `OpenAiChatModel` 构造自定义 `ChatClient`，不依赖自动装配。同理，`TranscriptionModel` 也可以手动构造。 [docs.spring](https://docs.spring.io/spring-ai/reference/api/chatclient.html)

```java
package com.example.demo.config;

import org.springframework.ai.chat.client.ChatClient;
import org.springframework.ai.openai.OpenAiChatModel;
import org.springframework.ai.openai.OpenAiChatOptions;
import org.springframework.ai.openai.api.OpenAiApi;
import org.springframework.ai.openai.OpenAiAudioTranscriptionModel;
import org.springframework.ai.openai.OpenAiAudioTranscriptionOptions;
import org.springframework.ai.audio.transcription.TranscriptionModel;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class AiConfig {

    // ─── 主模型（MiniMax M2.7，纯文本） ───

    @Value("${app.ai.main.base-url}")
    private String mainBaseUrl;

    @Value("${app.ai.main.api-key}")
    private String mainApiKey;

    @Value("${app.ai.main.model}")
    private String mainModel;

    // ─── 视觉模型（字节豆包，火山方舟） ───

    @Value("${app.ai.vision.base-url}")
    private String visionBaseUrl;

    @Value("${app.ai.vision.api-key}")
    private String visionApiKey;

    @Value("${app.ai.vision.model}")
    private String visionModel;

    // ─── 语音转写（智谱 GLM-ASR） ───

    @Value("${app.ai.transcription.base-url}")
    private String transcriptionBaseUrl;

    @Value("${app.ai.transcription.api-key}")
    private String transcriptionApiKey;

    @Value("${app.ai.transcription.model}")
    private String transcriptionModel;


    /**
     * 主 ChatClient：MiniMax M2.7，只处理文本，不传图片
     * 用于 RAG 检索后的最终回答生成
     */
    @Bean("mainChatClient")
    ChatClient mainChatClient() {
        OpenAiApi api = OpenAiApi.builder()
                .baseUrl(mainBaseUrl)
                .apiKey(mainApiKey)
                .build();

        OpenAiChatModel model = OpenAiChatModel.builder()
                .openAiApi(api)
                .defaultOptions(OpenAiChatOptions.builder()
                        .model(mainModel)
                        .build())
                .build();

        return ChatClient.create(model);
    }

    /**
     * 视觉 ChatClient：字节豆包 doubao-vision-pro-32k，接受图片输入
     * 只负责"看图说话"，输出图片的文字描述，不参与 RAG 和最终回答
     */
    @Bean("visionChatClient")
    ChatClient visionChatClient() {
        OpenAiApi api = OpenAiApi.builder()
                .baseUrl(visionBaseUrl)
                .apiKey(visionApiKey)
                .build();

        OpenAiChatModel model = OpenAiChatModel.builder()
                .openAiApi(api)
                .defaultOptions(OpenAiChatOptions.builder()
                        .model(visionModel)
                        .build())
                .build();

        return ChatClient.create(model);
    }

    /**
     * TranscriptionModel：智谱 GLM-ASR，语音转文字
     * 接口兼容 OpenAI /audio/transcriptions 风格
     */
    @Bean
    TranscriptionModel transcriptionModel() {
        OpenAiApi api = OpenAiApi.builder()
                .baseUrl(transcriptionBaseUrl)
                .apiKey(transcriptionApiKey)
                .build();

        return new OpenAiAudioTranscriptionModel(
                api,
                OpenAiAudioTranscriptionOptions.builder()
                        .model(transcriptionModel)
                        .responseFormat(
                                OpenAiAudioTranscriptionOptions.TranscriptResponseFormat.JSON)
                        .build()
        );
    }
}
```

## 第四步：SpringAiMultimodalChatService.java（核心逻辑改动）

这里是最重要的修改：**注入两个 ChatClient，图片先走视觉模型出文字描述，再把描述文字喂给主模型**。

```java
package com.example.demo.service.impl;

import com.example.demo.service.MultimodalChatService;
import com.example.demo.service.RagService;
import org.springframework.ai.audio.transcription.TranscriptionModel;
import org.springframework.ai.chat.client.ChatClient;
import org.springframework.ai.content.Media;
import org.springframework.beans.factory.annotation.Qualifier;
import org.springframework.core.io.Resource;
import org.springframework.http.MediaType;
import org.springframework.stereotype.Service;

@Service
public class SpringAiMultimodalChatService implements MultimodalChatService {

    private final ChatClient mainChatClient;      // MiniMax M2.7，只处理文本
    private final ChatClient visionChatClient;    // 豆包视觉，处理图片
    private final TranscriptionModel transcriptionModel;
    private final RagService ragService;

    public SpringAiMultimodalChatService(
            @Qualifier("mainChatClient") ChatClient mainChatClient,
            @Qualifier("visionChatClient") ChatClient visionChatClient,
            TranscriptionModel transcriptionModel,
            RagService ragService
    ) {
        this.mainChatClient = mainChatClient;
        this.visionChatClient = visionChatClient;
        this.transcriptionModel = transcriptionModel;
        this.ragService = ragService;
    }

    @Override
    public String chat(String query,
                       String instruction,
                       Resource image,
                       String imageContentType,
                       Resource audio) {

        StringBuilder userTextBuilder = new StringBuilder();

        // 1. 保留用户原始文本
        if (query != null && !query.isBlank()) {
            userTextBuilder.append("用户文本: ").append(query).append("\n");
        }

        // 2. 语音转写（GLM-ASR → 文字）
        if (audio != null) {
            String transcription = transcriptionModel.transcribe(audio);
            userTextBuilder.append("语音转写: ").append(transcription).append("\n");
        }

        // 3. 图片理解（豆包视觉 → 文字描述）
        //    关键：visionChatClient 只做"把图片转成文字描述"，不做 RAG，不做最终回答
        if (image != null) {
            String imageDescription = describeImage(image, imageContentType, query);
            userTextBuilder.append("图片内容描述: ").append(imageDescription).append("\n");
        }

        String userText = userTextBuilder.toString();
        if (userText.isBlank()) {
            userText = "用户未提供有效输入。";
        }

        // 4. RAG：基于综合后的文字内容做知识检索
        String context = ragService.buildContext(userText);

        // 5. 主模型（MiniMax M2.7）生成最终回复——只接收文本
        String systemPrompt = """
                你是一个知识库问答助手。请基于提供的知识上下文回答用户问题。
                用户的输入可能来自文字、语音转写或图片内容描述，请综合理解后作答。
                如果知识库中没有足够信息，请坦诚说明并给出合理推断。
                """;

        String finalUserText = """
                [知识上下文]
                %s

                [用户输入]
                %s
                """.formatted(context, userText);

        // mainChatClient 只传文本，不传 Media，对 MiniMax M2.7 友好
        return mainChatClient.prompt()
                .system(systemPrompt)
                .user(finalUserText)
                .call()
                .content();
    }

    /**
     * 使用视觉模型将图片转成文字描述。
     * 返回结果是一段文字，随后被融入 mainChatClient 的文本上下文。
     */
    private String describeImage(Resource image, String imageContentType, String hint) {
        MediaType mt = MediaType.IMAGE_JPEG;
        if (imageContentType != null && !imageContentType.isBlank()) {
            try {
                mt = MediaType.parseMediaType(imageContentType);
            } catch (Exception ignored) {}
        }

        Media media = new Media(mt, image);

        // 如果用户有文本提示，一并告诉视觉模型，让描述更有针对性
        String visionPrompt = (hint != null && !hint.isBlank())
                ? "用户问题是：" + hint + "\n请结合用户问题，详细描述图片内容，提炼出与问题最相关的视觉信息。"
                : "请详细描述这张图片的内容，包括文字、数据、图表、场景等所有重要信息。";

        return visionChatClient.prompt()
                .user(user -> user.text(visionPrompt).media(media))
                .call()
                .content();
    }
}
```
